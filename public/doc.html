Project Documentation

The project "FIWARE Demo" used several specific tools and frameworks tailored to develop a responsive and real-time urban mobility application. Hereâ€™s a description of each tool and its purpose in this project:

FIWARE Framework: This open-source framework for building smart applications was used to create the "Smart Transit Explorer." It allowed integration for urban mobility data and facilitated interaction between different components, specifically enabling smart city solutions.

Node.js: Serving as the backend, Node.js was crucial for processing requests and managing data flow between the Wiener Linien API and Orion LD. It dynamically handled API requests, providing real-time updates to the frontend.

Leaflet.js: This JavaScript library for interactive maps was used in the frontend to display real-time public transport data on a map interface, enhancing the user experience by providing a visual and interactive component.

Orion LD Context Broker: This component of the FIWARE ecosystem enabled the handling and querying of real-time context data, which was necessary for managing live updates in public transport data, improving both data accuracy and system responsiveness.

Docker: Used for containerization, Docker ensured that the development and deployment environments were consistent, making it easier to deploy the application across different platforms without compatibility issues.

GitLab and Git: For version control and collaboration, GitLab and Git were essential. They provided a repository tracking code changes and supporting collaboration among team members through shared repositories.
